# **Security Policy**

## **Supported Versions**

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1.0 | :x:                |

---

## **Reporting a Vulnerability**

* Please report vulnerabilities through the **GitHub Issues** page of this repository.
* When creating an issue, use the **"Security Report"** template (if available) or clearly label the issue as a vulnerability.
* You will receive an acknowledgement within **2-5 days**.
* You will receive further updates every **3–7 days** until the issue is resolved.
* Once validated, the vulnerability will be:

  * Assigned a severity level
  * Patched in the next minor or patch release
  * Included in the project’s changelog
* If a report is declined, you will be provided with a clear explanation.

Please do **not** disclose security issues publicly until a fix is released.
